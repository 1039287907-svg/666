# 1 项目概述

## 1.1 项目背景

建筑火灾是威胁公共安全的主要灾害之一，建筑材料的防火性能直接决定了火灾的蔓延速度和人员逃生时间。ISO 11820"着火试验—建筑材料—不燃性试验"是国际通用的建筑材料不燃性检测标准，通过将材料样品置于750°C加热炉中60分钟，记录温度变化和失重率，判定材料是否属于"不燃材料"。该标准在全球建筑工程质量监管中具有重要地位。

当前，建筑材料防火检测教学面临着设备昂贵、试验周期长、安全风险高、教学效率低等突出困难。一套标准的不燃性试验设备价格在20-50万元人民币，且每年需要专业维护和检定，使得大多数高校只能配备极少量设备，难以满足教学需求。同时，750°C的高温操作给学生实训带来烫伤和火灾风险。在此背景下，开发一套纯软件仿真的试验系统，以低成本、零风险的方式替代真实试验操作，具有重要的现实意义。

## 1.2 项目目标

本项目的核心研究目标是：在无需真实物理硬件的前提下，构建一套完整的、符合ISO 11820标准的建筑材料不燃性试验仿真软件系统。具体研究目标包括：

（1）建立可信的5通道温度仿真数学模型，使仿真生成的温度数据在物理规律上合理可信。

（2）设计严格的试验状态机模型，准确模拟真实试验过程中的各种状态及其转换条件。

（3）实现实时数据可视化和交互控制，提供接近真实试验设备的操作体验。

（4）实现标准格式报告的自动生成，满足教学管理和标准化检测的需求。

## 1.3 项目意义

（1）学术意义：探索了将物理试验过程数字化的方法，提出了一种面向教学的仿真系统设计范式，特别是温度仿真中基于热传导原理的多通道建模方法和基于线性回归的智能稳定判定方法，对其他工程试验仿真系统的开发具有参考价值。

（2）应用意义：为高校建材防火课程提供了实用的教学工具，使学生能够在不依赖昂贵硬件设备的情况下获得充分的动手实践机会。

（3）社会意义：有助于推动建筑材料防火检测标准的普及和检测人才的培养，从源头上服务于建筑消防安全水平的提升。

# 2 国内外研究现状

## 2.1 建筑材料不燃性检测技术

ISO 11820标准自发布以来，经历了多个版本的修订。国际上，欧洲、北美和日本的检测机构广泛采用该标准进行建筑材料的合规性判定。在检测设备方面，德国NETZSCH、美国TA Instruments等厂商提供了成熟的商业化试验炉和配套软件，但这些系统均为"硬件+软件"的捆绑方案，价格昂贵，且不提供脱离硬件的独立仿真模式。

在国内，GB/T 5464-2010《建筑材料不燃性试验方法》等效采用了ISO 11820的技术内容。国内多家检测设备制造商（如北京建筑材料检验研究院、上海建科院等）也开发了相应的检测设备，但同样面向真实试验场景，未见专门的教学仿真软件产品。

## 2.2 虚拟仿真实验教学

教育部自2013年起推动虚拟仿真实验教学中心建设，2017年进一步启动了虚拟仿真实验教学项目建设工作。在此政策推动下，各高校在土木工程、化学化工、机械工程等领域开发了大量的虚拟仿真实验教学项目。建材防火领域的虚拟仿真也取得了一些进展，如部分高校开发了基于Unity WebGL的建筑材料燃烧性能虚拟仿真实验。

然而，现有建材防火虚拟仿真项目多为Web端实现，存在以下局限：（1）操控真实感不足，Web端的交互延迟和UI表现与桌面原生应用存在差距；（2）仿真模型粗糙，多数项目采用简单的线性插值生成温度数据，不体现热传导物理规律；（3）缺乏完整的试验流程管理和标准报告生成功能，仅关注单一的操作演示。

## 2.3 现有研究的不足

综合国内外研究现状，当前存在以下研究空白：

（1）缺乏面向ISO 11820标准的专门教学仿真软件，市场上尚无覆盖完整试验流程的桌面级仿真产品。

（2）温度仿真模型过于简化，多数项目未考虑多通道温度之间的物理关系（炉温→表面温→中心温的热传导链）。

（3）缺乏严格的试验状态机建模，现有仿真系统往往未对状态转换条件进行严格约束，导致操作逻辑与真实设备差距较大。

（4）缺乏标准化的报告自动生成能力，教学场景中需要手动整理试验数据并撰写报告，效率低下。

本项目正是针对以上研究空白，开展面向ISO 11820标准的建筑材料不燃性试验仿真系统研究。

# 3 项目研究内容

## 3.1 温度仿真模型研究

本研究建立了基于热传导基本原理的5通道温度仿真模型。不同于简单的随机数生成或线性插值，本模型将5个温度通道分为三个物理层次：

第一层——热源层（炉温1 TF1、炉温2 TF2）：模拟加热炉内的温度变化。在升温阶段以线性速率上升至目标温度（750°C），在稳定阶段通过钳位+随机噪声维持在目标温度附近。两个通道使用独立随机噪声源，模拟双传感器冗余测量的真实场景。

第二层——表面传导层（表面温度 TS）：模拟样品表面的温度变化。在记录阶段采用指数趋近模型 TS(t+1) = TS(t) + [min(TF1×0.95, 800) - TS(t)] × 0.02 + noise，体现表面温度逐步趋近炉温的热传导过程。

第三层——中心传导层（中心温度 TC）：模拟样品中心的温度变化。采用与表面温相同形式但参数不同的指数趋近模型 TC(t+1) = TC(t) + [min(TF1×0.85, 750) - TC(t)] × 0.01 + noise。0.01的趋近系数小于表面温的0.02，体现了热能从表面传导到中心所需的额外时间延迟。

该模型的物理合理性体现在：任意时刻均满足 TC < TS < TF1（中心温 < 表面温 < 炉温），且表面温上升速度快于中心温，符合热传导从外到内的基本规律。

## 3.2 试验状态机研究

本研究将ISO 11820试验的操作流程抽象为5状态有限状态机（Idle → Preparing → Ready → Recording → Complete），严格定义了7种状态转换路径及各自的触发条件、前置约束和附加动作。

状态机的核心设计要点包括：

（1）自动转换条件：Preparing→Ready的转换由系统基于温度稳定性判定自动触发，不依赖人工判断，体现了智能化特征。

（2）回退保护机制：Ready状态下若温度跌出稳定范围（745~755°C），自动回退到Preparing状态，防止在不稳定状态下开始记录。

（3）数据保护锁：当检测到存在"已完成但未保存"的试验记录（flag != "10000000"）时，系统全局锁定新建试验和开始记录操作，强制用户先保存当前结果，防止数据丢失。

（4）恒功率计算：Ready状态下持续记录PID输出值到队列，开始记录时取队列平均值作为恒功率值，模拟真实设备中从PID温控切换到恒功率模式的过渡过程。

该状态机的设计确保了软件在任何操作序列下行为确定且可预测，有效防止了用户的误操作。

## 3.3 实时数据可视化研究

本研究实现了一套基于事件驱动架构的实时数据可视化方案。后台DaqWorker线程以800ms为周期生成仿真数据并通过DataBroadcast事件广播，UI线程通过Invoke机制安全地接收事件数据并更新界面控件。

可视化的核心设计要素包括：

（1）LED风格温度数值：采用大号等宽字体（Consolas）在黑底上显示绿色温度数字，模拟工业仪表盘的视觉效果，保留1位小数，辨识度高。

（2）4通道实时曲线图：使用OxyPlot的高性能渲染引擎，4条折线以红/蓝/绿/橙4色区分，X轴为滚动时间窗（最近10分钟），Y轴固定为0~800°C。采用增量更新策略（仅追加新数据点），避免全量重绘导致的闪烁。

（3）系统消息日志：关键事件以带时间戳（HH:mm:ss格式）的消息形式实时滚动输出，不同级别的事件以不同颜色标记（白色=普通，黄色=警告/终止），帮助用户快速识别重要状态变化。

## 3.4 标准化报告自动生成研究

本研究实现了试验完成后的CSV/Excel/PDF三种格式报告一键自动生成。

Excel报告（EPPlus实现）：3个Sheet结构——Sheet1=试验信息表（含样品信息、试验参数、质量数据、各通道温度统计值和判定结论）、Sheet2=从CSV文件读取的完整温度数据明细（每秒一行）、Sheet3=使用EPPlus内置Chart功能生成的4通道温度曲线图。

PDF报告（PDFsharp-MigraDoc实现）：采用文档模型（Document Object Model）构建PDF内容——标题→试验概要表格（MigraDoc.Table）→温度曲线位图（OxyPlot截图嵌入）→判定结论段落。

判定结论采用简化规则：deltatf（样品温升）≤ 50°C 且 lostweight_per（失重率）≤ 50% 且 flameduration（火焰持续时间）< 5秒 →"通过"，否则"不通过"。该规则简化自ISO 11820标准的完整判定准则，适用于教学演示场景。

# 4 研究方法与技术路线

## 4.1 研究方法

本项目采用以下研究方法：

（1）文献研究法：系统收集和分析了ISO 11820标准文本、建筑材料不燃性试验相关研究论文、虚拟仿真实验教学政策文件和技术文献，为系统设计提供理论和规范依据。

（2）模型构建法：采用有限状态机理论建立试验状态转换模型，采用数值分析方法（线性回归）实现温度稳定性的智能判定，采用指数趋近方程模拟热传导过程中的温度变化。

（3）分层设计法：将系统按照界面层、业务核心层、服务层、数据层四层架构进行分层设计，遵循"上层依赖下层、下层不感知上层"的软件工程原则。

（4）实验验证法：通过端到端的全流程测试验证系统的功能完整性，通过长时间运行测试验证系统的性能稳定性，通过多场景测试验证系统的兼容性。

## 4.2 技术路线

第1阶段——需求分析与方案设计：调研ISO 11820标准要求、分析教学场景痛点、明确功能需求和非功能需求、设计系统架构和模块划分。

第2阶段——基础设施搭建：建立数据模型（6个实体类）、实现数据库访问层（DbHelper）、完成配置管理和日志系统、构建全局上下文单例。

第3阶段——核心业务实现：开发温度仿真引擎（SensorSimulator）、实现试验状态机（TestController）、建立数据采集调度（DaqWorker）、实现温漂计算和终止判定。

第4阶段——界面与服务开发：开发登录和主界面窗体、集成OxyPlot实时曲线、实现Excel和PDF报告生成、开发历史查询和设备校准功能。

第5阶段——集成测试与优化：端到端流程测试、Bug修复、性能优化、文档完善。

## 4.3 关键技术突破

（1）多通道温度仿真模型：基于热传导基本原理，首次在ISO 11820仿真领域实现了5通道温度数据的物理合理生成，特别是表面温和中心温的差异化指数趋近模型，真实体现了从表面到中心的热传导延迟。

（2）基于线性回归的智能温度稳定判定：采用MathNet.Numerics对最近10分钟温度序列进行线性回归，以拟合斜率作为温漂指标，相比传统阈值法能更有效地平滑随机噪声，提供更准确的稳定性判断。

（3）仿真/硬件双模式可切换架构：数据采集层通过配置开关在仿真模式和Modbus硬件模式之间切换，同一套代码同时支持教学仿真和实际检测，实现了从教学到生产的平滑迁移路径。

（4）面向AI辅助开发的文档驱动设计：开发文档以"面向AI理解"为目标编写，包含完整的功能规格、算法伪代码、数据库设计、分工方案等，使得其他开发者可使用AI编程工具自行复现本项目。这一方法为软件工程教学提供了一种新的范式。

# 5 研究成果

## 5.1 软件成果

本项目的核心软件成果是一套功能完整的ISO 11820建筑材料不燃性试验仿真Windows桌面应用程序，具备以下功能特性：

（1）基于角色的登录认证（管理员/试验员两种角色）。

（2）完整的试验流程管理（新建试验→升温控制→温度稳定自动判定→记录控制→试验现象记录→数据保存）。

（3）5通道温度仿真（炉温1、炉温2、表面温度、中心温度、校准温度），仿真参数可配置。

（4）实时数据可视化（LED风格温度显示 + 4通道实时曲线图 + 系统消息日志）。

（5）一键式报告自动生成（CSV原始数据文件 + Excel报告含内嵌图表 + PDF报告）。

（6）历史试验记录的多条件组合查询和详情查看。

（7）设备校准管理（9点炉壁温度校准 + 中心轴扫描校准 + 历史追溯）。

（8）可配置的系统参数（仿真参数、文件路径等通过配置文件管理）。

## 5.2 文档成果

本项目产出以下技术文档：

（1）ISO11820-开发文档.md：面向AI辅助开发的功能规格文档，涵盖软件定位、功能需求、技术栈、仿真算法、代码分层结构等核心内容。

（2）DB-数据库设计.md：完整的数据库设计文档，包含6张表的建表SQL、字段说明、初始数据、关系图和使用示例。

（3）ISO11820-三人分工方案.md：面向三人协作开发的详细分工方案，定义了Person A/B/C的职责范围、任务清单和接口约定。

（4）宣传视频录屏脚本.md：宣传视频的完整脚本，包含操作步骤和配文说明。

（5）项目汇报PPT（ISO11820_仿真系统_项目汇报.pptx）。

## 5.3 创新成果

（1）首次以桌面原生应用的形式实现了ISO 11820完整试验流程的仿真，填补了该细分领域的软件空白。

（2）建立了基于热传导原理的5通道温度仿真模型，表面温和中心温的差异化指数趋近模型具有物理可信度。

（3）提出了基于线性回归的智能温度稳定判定方法，相较于传统阈值比较法更准确和鲁棒。

（4）实现了仿真/硬件双模式可切换架构，为教学软件向工程实用软件的演进提供了技术基础。

（5）提出了面向AI辅助开发的文档驱动设计方法，为同类教学软件开发提供了一种可复制的新范式。

# 6 应用前景与社会价值

## 6.1 应用前景

（1）高校教学：可直接应用于建筑材料、消防工程、安全工程等专业的课堂教学和学生自主练习，服务于全国数十所相关高校的数千名学生。

（2）检测机构培训：可用于建筑材料检测机构的新入职人员岗前培训和标准流程操作考核。

（3）标准宣贯推广：可用于ISO 11820和GB/T 5464标准的宣贯培训，帮助从业人员深入理解试验方法和判定准则。

（4）科研算法验证：仿真模式下可快速调整参数进行大量重复试验，为PID温控算法、材料热性能预测模型等研究提供仿真验证环境。

（5）未来拓展——对接真实硬件：基于双模式架构，未来可通过接入Modbus传感器和PID控制器，将系统升级为实际检测设备的控制软件，实现从教学仿真实训到工程实际应用的全覆盖。

## 6.2 社会价值

（1）降低教育门槛：将建材防火试验教学的硬件门槛从数十万元降至几乎为零，推动优质教育资源的普及。

（2）保障教学安全：用仿真替代高温试验操作，从根源上消除学生实训中的烫伤和火灾风险。

（3）提升人才培养质量：增加学生动手实践的机会，培养更多掌握建筑材料防火检测技术的高素质专业人才。

（4）服务建筑消防安全：通过提升检测人才培养质量，间接服务于建筑消防安全整体水平提升，保障人民生命财产安全。

# 7 总结与展望

本项目成功开发了一套功能完整、界面友好、部署简便的ISO 11820建筑材料不燃性试验仿真系统。系统以C#/.NET 8为开发平台，采用WinForms+SQLite技术栈，建立了基于热传导原理的5通道温度仿真模型，设计了严格的5状态试验状态机，实现了实时数据可视化和标准化报告自动生成，填补了面向教学的ISO 11820试验仿真软件的市场空白。

项目的主要贡献包括：（1）提出了基于热传导基本原理的5通道温度仿真方法；（2）设计了基于线性回归的智能温度稳定判定算法；（3）实现了仿真/硬件双模式可切换的软件架构；（4）探索了面向AI辅助开发的文档驱动设计新范式。

未来工作方向：

（1）硬件对接：基于仿真/硬件双模式架构，接入真实Modbus传感器和PID控制器，实现从教学仿真到实际检测的升级。

（2）多炉支持：将单炉架构扩展为多炉同时试验的架构，满足大型实验室的教学需求。

（3）云端管理：开发Web端数据管理平台，实现多用户、多班级的教学数据统一管理和学情分析。

（4）AI融合：引入机器学习模型，基于历史数据训练更准确的试验结果预测和异常检测能力。

（5）国际化：增加多语言界面支持，拓展海外用户群体。
