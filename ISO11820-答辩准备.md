# ISO 11820 建筑材料不燃性试验仿真系统 — 答辩准备

## 一、项目概述（1 分钟）

**一句话**：用 C# WinForms 仿真的 ISO 11820 建筑材料不燃性试验系统，无需真实硬件即可完整演示试验流程。

| 项目信息 | 说明 |
|---------|------|
| 项目名称 | ISO 11820 建筑材料不燃性试验仿真系统 |
| 开发平台 | .NET 8 / Windows Forms |
| 数据库 | SQLite（本地文件） |
| 代码规模 | 26 个源文件，约 4600 行 C# 代码 |
| 第三方库 | OxyPlot（图表）、EPPlus（Excel）、PDFsharp（PDF）、Serilog（日志）、MathNet（数学） |

## 二、系统架构（2 分钟）

### 2.1 分层架构

```
┌──────────────────────────────────────┐
│          UI 层（Forms）              │
│  LoginForm / MainForm / Dialogs     │
│  ChartPanel（独立图表控件）          │
├──────────────────────────────────────┤
│         业务核心层（Core）            │
│  TestController（5 状态机）          │
│  SensorSimulator（5 通道仿真引擎）   │
│  DaqWorker（数据采集调度）           │
├──────────────────────────────────────┤
│         数据层（Data）               │
│  DbHelper（SQLite CRUD 封装）       │
├──────────────────────────────────────┤
│         基础设施层                    │
│  Models / AppConfig / AppContext     │
└──────────────────────────────────────┘
```

**核心设计原则**：
- 上层依赖下层，下层通过**事件（event）**向上传递数据
- 所有 UI 更新通过 `Invoke` 切回 UI 线程
- AppContext 单例持有全局对象，各层通过它获取依赖

### 2.2 三人分工

| 人员 | 层级 | 核心文件 |
|------|------|---------|
| **Person A** | 基础设施 | Models(6)、DbHelper、AppConfig、AppContext、Program |
| **Person B** | 业务核心 | TestController、SensorSimulator、DaqWorker、状态机 |
| **Person C** | UI + 导出 | MainForm(4 partial)、ChartPanel、LoginForm、Dialog×3、ExportService |

依赖顺序：A → B → C

## 三、核心技术亮点（3 分钟）

### 3.1 5 状态机

```
Idle（空闲）→ Preparing（升温）→ Ready（就绪）→ Recording（记录）→ Complete（完成）
```

- 状态转换由 `TestController` 严格管控
- 按钮可用性随状态自动联动（6 个按钮 × 5 个状态 = 30 个条件）
- Ready 状态下温度跌出范围自动回退 Preparing

### 3.2 5 通道温度仿真引擎

| 通道 | 行为 |
|------|------|
| 炉温1（TF1） | 升温至 750°C 稳定，含随机噪声 ±0.5°C |
| 炉温2（TF2） | 与 TF1 独立噪声 |
| 表面温（TS） | Recording 阶段向 TF1×0.95 指数逼近 |
| 中心温（TC） | Recording 阶段向 TF1×0.85 指数逼近 |
| 校准温（TCal）| = TF1 + 随机 ×2 |

- 每 800ms 一帧，仿真升温 / 稳定 / 记录 / 降温四个阶段
- 稳定判定：连续 4 帧（3.2 秒）在 745~755°C 范围内

### 3.3 实时数据可视化（OxyPlot）

- 4 条曲线同框（炉温1/2、表面温、中心温），各配独立颜色
- Y 轴自适应范围，X 轴滚动显示最近 600 个数据点
- 支持鼠标缩放/拖拽/框选
- Recording 前样品温度曲线自动隐藏（NaN 占位）

### 3.4 温漂计算

- Recording 阶段持续追踪炉温历史（最多 600 个数据点）
- 用最小二乘法线性回归计算温漂（°C/10min）
- 温漂达标时触发试验自动终止

### 3.5 报告导出

- **CSV**：每秒温度数据自动保存
- **Excel**：3 Sheet（试验信息 + 温度数据 + 图表）
- **PDF**：自定义字体解析器嵌入中文（SimHei），直接绘制文本和表格

### 3.6 判定逻辑

```
通过条件：炉温升 ≤ 30°C  且  失重率 ≤ 50%  且  火焰持续 ≤ 20 秒
```

## 五、可能被问的问题（Q&A 准备）

### Q1：为什么状态机从 Complete 要回到 Idle 而不是 Preparing？
完成并保存后炉子自然冷却，下次试验重新升温，更符合真实流程。

### Q2：仿真温度算法的依据是什么？
严格按开发文档第 2.3 节，升温/稳定/记录三个阶段各有独立的温度更新公式，升温速率和波动幅度从 appsettings.json 可配置。

### Q3：为什么用了 PDFsharp 而不是 MigraDoc 生成 PDF？
MigraDoc 对中文字体内嵌支持不足。PDFsharp 通过自定义 IFontResolver 从 Windows 字体目录直接加载 simhei.ttf，确保中文正确渲染。

### Q4：OxyPlot 的图例为什么用手绘而不是内置的？
OxyPlot 2.2.0 的 Legend 类与 WinForms 渲染器存在兼容问题，改用 WinForms 原生 Label + Panel 绘制，稳定可靠且颜色与曲线一致。

### Q5：校准 Tab 中的 9 点测温是什么？
模拟 ISO 标准中炉温均匀性标定：A/B/C 三层 × 1/2/3 轴共 9 个测温点，计算平均值和最大偏差（≤10°C 判定通过）。

### Q6：三个人如何协作？接口如何约定？
- A 提供 AppContext.Instance.Db 和 AppContext.Instance.Config
- B 通过 DataBroadcast 事件向 C 推送数据
- C 直接调用 DbHelper 进行数据读写
- 依赖顺序严格 A→B→C，后写的先测试前一层接口

### Q7：如果硬件就绪，如何切换到真实采集？
DaqWorker 中 EnableSimulation 配置项控制：仿真模式走 SensorSimulator，硬件模式走串口 Modbus 读取。两套逻辑接口一致，切换只需改配置。
