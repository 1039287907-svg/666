# 1 引言

## 1.1 编写目的

本文档旨在对ISO 11820建筑材料不燃性试验仿真系统进行全面的需求分析，明确系统的功能需求、非功能需求、数据需求和接口需求。本文档是后续概要设计、详细设计和测试工作的基础依据。预期读者包括：开发团队成员、指导教师、项目评审专家。

## 1.2 项目背景

ISO 11820是国际通用的建筑材料不燃性检测标准。在高校教学和检测机构培训场景中，真实试验设备价格昂贵（单套数十万元）、试验周期长（60分钟/次）、高温操作（750°C）存在安全隐患、数据管理手段落后。为解决这些痛点，开发一套纯软件仿真的ISO 11820不燃性试验系统，无需真实硬件即可完整体验标准试验流程。

## 1.3 术语定义

| 术语 | 说明 |
| --- | --- |
| TF1 / 炉温1 | 加热炉内温度传感器1的测量值，系统的主要温度参考通道 |
| TF2 / 炉温2 | 加热炉内温度传感器2的测量值，双传感器冗余测量 |
| TS / 表面温 | 试验样品表面温度 |
| TC / 中心温 | 试验样品中心温度 |
| TCal / 校准温 | 设备校准用温度通道，仅数值显示，不参与曲线绘制 |
| Preparing | 升温中状态，炉温从当前值上升到目标温度的阶段 |
| Ready | 就绪状态，炉温已达到目标温度且保持稳定，可开始记录 |
| Recording | 记录中状态，正在逐秒记录温度数据 |
| Complete | 完成状态，试验数据采集已结束，等待用户保存试验记录 |
| 失重率 | （试验前质量 - 试验后质量）/ 试验前质量 × 100%，判定项 |
| 温漂 | 最近10分钟炉温序列线性回归的斜率，用于判定温度稳定性，单位°C/10min |
| 恒功率 | Ready阶段PID输出值的平均值，记录阶段使用该值维持炉温 |

# 2 项目概述

## 2.1 项目目标

开发一款运行于Windows 10/11平台的桌面应用程序，实现以下核心目标：

（1）提供ISO 11820标准试验的完整流程仿真，涵盖从试验创建到报告生成的全部环节。

（2）建立基于热传导原理的5通道温度仿真模型，模拟炉温升温、稳定、表面温和中心温的热传导特性。

（3）实现实时数据可视化，包括5通道温度数值LED显示、4通道温度曲线图、系统消息日志。

（4）支持一键生成CSV原始数据文件、Excel报告（含内嵌图表）和PDF报告。

（5）支持历史试验记录的多条件查询、详情查看和数据重新导出。

## 3.3 温度仿真与实时监控（FR-SIM）

| 需求编号 | 需求描述 | 优先级 | 验收标准 |
| --- | --- | --- | --- |
| FR-SIM-001 | 系统仿真生成5通道温度数据：炉温1(TF1)、炉温2(TF2)、表面温(TS)、中心温(TC)、校准温(TCal) | 高 | 5通道数据每800ms更新一次 |
| FR-SIM-002 | 炉温1和炉温2在升温阶段按配置速率线性上升，到达747°C以上进入稳定阶段钳位到750°C+噪声 | 高 | 升温速率由HeatingRatePerSecond参数控制，稳定温度=TargetFurnaceTemp+噪声 |
| FR-SIM-003 | 炉温1和炉温2使用独立随机噪声源，体现双传感器冗余测量 | 中 | TF1和TF2在任意时刻数值不同但接近 |
| FR-SIM-004 | 表面温度在记录阶段以指数趋近模型向min(TF1×0.95, 800)接近 | 高 | TS数值随时间逐步上升，始终满足TC < TS < TF1 |
| FR-SIM-005 | 中心温度在记录阶段以比表面温更慢的指数趋近模型向min(TF1×0.85, 750)接近 | 高 | TC上升速度慢于TS，体现热传导从表面到中心的延迟 |
| FR-SIM-006 | 主界面以LED风格大字显示5通道温度数值（保留1位小数，单位°C） | 高 | 数值每秒刷新，字体清晰醒目 |
| FR-SIM-007 | 主界面以OxyPlot实时绘制4条温度曲线（TF1/TF2/TS/TC），4色区分，X轴滚动显示最近10分钟 | 高 | 曲线流畅无闪烁，校准温不在曲线图中显示 |
| FR-SIM-008 | 主界面显示系统消息日志，不同事件以不同颜色标记 | 高 | 消息包含时间戳（HH:mm:ss）和内容，关键事件黄色标记，自动滚动到最新 |
| FR-SIM-009 | 主界面显示当前试验状态中文描述、计时器（仅Recording累加）、温度漂移值 | 高 | 状态文字颜色随状态变化，计时器精确到秒，温漂单位°C/10min |

# 5 数据需求

## 5.1 数据实体

系统包含以下核心数据实体：

| 实体 | 存储方式 | 说明 |
| --- | --- | --- |
| 操作员 (Operator) | SQLite: operators表 | 包含userid, username, pwd, usertype，用于登录认证 |
| 设备 (Apparatus) | SQLite: apparatus表 | 包含设备编号、名称、检定日期、串口配置、恒功率值，全局仅1条记录 |
| 样品 (ProductMaster) | SQLite: productmaster表 | 包含样品编号(PK)、名称、规格、尺寸 |
| 试验记录 (TestMaster) | SQLite: testmaster表 | 核心实体，联合主键(productid, testid)，约40个字段，包含基本信息、质量数据、各通道温度统计、计算结果 |
| 传感器 (Sensor) | SQLite: sensors表 | 包含传感器编号、名称、量程、当前值，共17个通道 |
| 校准记录 (CalibrationRecord) | SQLite: CalibrationRecords表 | 包含校准日期、类型、9点温度数据(JSON)、统计结果 |
| 温度时序数据 | CSV文件 | 不入库，每次试验独立一个CSV文件，每秒一行6列(Time+5通道温度) |

## 5.2 数据流

系统内主要数据流如下：

（1）仿真温度数据流：SensorSimulator → DaqWorker → (a)更新sensors表outputvalue → (b)DataBroadcast事件 → MainForm UI刷新 → (c)Recording时写入CSV文件。

（2）试验管理数据流：MainForm → TestController → (a)DbHelper → SQLite数据库 → (b)StateChanged事件 → MainForm按钮状态更新。

（3）报告生成数据流：ExportService → (a)读取CSV文件 → Excel/PDF报告 → (b)读取testmaster表 → 报告中的试验信息表。

（4）配置数据流：appsettings.json → AppConfig → AppContext.Instance.Config → 各模块读取。

# 7 约束与假设

技术约束：

（1）开发平台为.NET 8，目标框架net8.0-windows，运行环境为Windows 10/11 x64。

（2）UI框架为Windows Forms，不使用WPF或MAUI。

（3）数据库为SQLite，不使用SQL Server或其他数据库服务。

（4）所有SQL操作使用Microsoft.Data.Sqlite直接编写参数化SQL，不使用EF Core ORM。

业务假设：

（1）系统为单用户单机使用，不涉及并发控制和多用户同时操作。

（2）试验仅支持单炉操作，不支持多炉同时试验。

（3）密码明文存储可接受（教学演示场景，不涉及隐私保护法规）。

（4）仿真模式下不执行真实的ISO标准合规判定（简化处理）。

（5）用户具备基本的Windows操作能力和建材防火试验基础知识。
